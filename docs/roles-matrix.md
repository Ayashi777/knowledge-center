# Матриця доступів (Roles × Permissions)

Ця матриця описує **рекомендовану** політику доступів. Вона не зафіксована жорстко в коді — реальний доступ визначається `viewPermissions` і `downloadPermissions` у документах.

Позначення:
- **V** — рекомендований доступ на перегляд
- **D** — рекомендований доступ на завантаження

## 1) Типові категорії контенту

| Категорія контенту | guest | foreman | engineer | architect | employee | worker | dispatcher | hr | admin |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Публічні оглядові матеріали | V | V | V | V | V | V | V | V | V/D |
| Монтажні інструкції |  | V/D | V/D | V | V | V/D | V |  | V/D |
| Інженерні специфікації |  |  | V/D | V | V |  |  |  | V/D |
| Архітектурні/фасадні рішення |  |  | V | V/D | V |  |  |  | V/D |
| Внутрішні регламенти |  |  | V | V | V/D |  | V | V | V/D |
| Логістика/графіки |  |  |  |  | V |  | V/D |  | V/D |
| HR політики/документи |  |  |  |  |  |  |  | V/D | V/D |

## 2) Як це застосувати в системі

- Для публічних документів: залишати `viewPermissions` порожнім.
- Для приватних: задавати `viewPermissions`.
- Для більш жорсткого контролю скачування: задавати `downloadPermissions`.

## 3) Нагадування

- **Admin** завжди має повний доступ (view + download).
- Якщо `downloadPermissions` порожній → він успадковує доступ від `viewPermissions`.
